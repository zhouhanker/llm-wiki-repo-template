# LLM-Wiki Schema

## 目的
本文件规定 wiki 页面最低字段，保证页面能被人和 Agent 稳定读取、维护和升级。

## 通用字段
所有 wiki 页面应尽量包含：
- 页面类型
- 当前状态
- 来源
- 更新时间
- 一句话摘要或定义
- 相关页面
- 待验证问题

## Source 页面
用于记录 raw 资料的来源和提炼结果。

建议结构：
```markdown
# Source: 标题

## 元信息
- 页面类型：Source
- 原文：
- 标题：
- 作者：
- 来源：
- 创建时间：
- 当前状态：

## 一句话摘要

## 关键观点

## 可复用结论

## 相关页面

## 待验证问题
```

## Concept 页面
用于解释一个概念、边界和当前版本理解。

建议结构：
```markdown
# 概念名

## 页面信息
- 页面类型：Concept
- 当前状态：
- 来源：
- 更新时间：

## 一句话定义

## 核心问题

## 适用范围

## 常见误解

## 相关页面

## 待验证问题
```

用于沉淀可复用流程、模板、检查清单或实践方法。

建议结构：
```markdown
# 模式名

## 页面信息
- 页面类型：Pattern
- 当前状态：
- 来源：
- 更新时间：

## 一句话定义

## 适用场景

## 操作流程

## 验收标准

## 常见风险

## 相关页面

## 待验证问题
```

## Project 页面
用于记录具体项目状态和任务接续信息。

建议结构：
```markdown
# 项目名

## 页面信息
- 页面类型：Project
- 当前状态：
- 负责人：
- 更新时间：

## 当前目标

## 已完成

## 进行中

## 阻塞项

## 关键决策

## 验证证据

## 下一步

## 相关页面
```

## 写入规则
- Source 页面应能回到 raw 原文或外部来源。
- Concept 和 Pattern 页面应引用对应 Source 页面。
- Project 页面应保留当前状态、决策和下一步，方便新会话接续。
- 稳定结论写入 wiki/；临时交付写入 output/；原始证据保留 raw/。
